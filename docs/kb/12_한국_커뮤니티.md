### 2. 엔진 — Unity (한글패치 메커니즘의 핵심)

- 본작은 **Unity 엔진** 기반으로 보인다. 근거: ① 팬위키(ai-seo) 다수가 Unity로 표기, ② **DCInside 한글패치가 `Sand_Data\data.unity3d`(Unity 에셋 번들)를 교체**하는 방식이라는 점(폴더명 `Game명_Data`, 파일 `data.unity3d`는 전형적 Unity 빌드 구조). [커뮤니티/ai-seo 교차]

### 4. 자동 한글패치(팬 제작) — 제작·배포·손상·밴 리스크

갤러리에 **익명 유저가 만든 기계번역(AI) 한글패치**가 1세대→현행으로 갱신되며 유통 중. 한국어 정식 미지원에 대한 사실상 유일한 대안. [커뮤니티]

- **#7 (06.16, 1세대)**: `Sand_Data` 폴더의 **`data.unity3d`를 받아 덮어쓰기**. 안내문 `왠만해서는 안걸리게 제작해놨으나 안티치트에 의해 0.1% 확률로 밴당할 가능성이 있다.` → **BattlEye 밴 가능성 0.1% 자체 추정치**를 명시. 구글 드라이브 배포. [커뮤니티]
- **#23 (06.23, 출시버전 빌드 업데이트)**: `한글패치 업데이트 (출시버전 빌드)`(조회 259). 기존 패치가 정식 출시 빌드와 호환되지 않아 **AI로 재생성**. 절차: 원본 `data.unity3d` 백업(이름변경) → 새 파일 복사 → 게임 내 `설정 → 언어 → 한국어` 선택. 갤러리 추출상 **기존 영어 슬롯 자리를 한국어로 덮는** 형태로 보이나, "영어 슬롯 교체"와 "한국어 선택"이 함께 언급돼 슬롯 지정은 **불명확/검증 필요**. [커뮤니티, 일부 불확실]
- **#25 (06.23, 현행 고정)**: `SandKoreanPatch_KR.zip`(구글 드라이브) 배포, **압축 해제 후 `Installer.bat` 실행**으로 적용(수동 복사 → 인스톨러 방식으로 진화). 경고 `기존이랑 동일하게 밴 당할 확률 존재`. [커뮤니티]
- **손상/글자 깨짐 (#30, 06.23 `한글자동패치 원래 글자 깨져나옴?`)**: 설치 후 `설치했더니 뜡꿕?? 이렇게 뜨고 설치뒤엔 게임실행도안되네` — **글자 깨짐 + 실행 불가** 보고. 댓글 0(미해결). 폰트 미포함/한글 글리프 부재로 추정되나 갤러리 내 해결 정보 없음. [커뮤니티]
- **수요 신호 (#22 `한글 없는데 어케 하고있어요?`)**: 한국어 부재 상황에서의 플레이 방법 문의. [커뮤니티]

> 요약: SAND 한글패치는 ① **AI 기계번역** ② **Unity `data.unity3d` 번들 교체** ③ **구글 드라이브 + `Installer.bat`** ④ **BattlEye 게임이라 온라인 플레이 시 밴 리스크 "본인 책임"** ⑤ **일부 환경에서 글자 깨짐/실행 불가**. 제작자는 익명(IP 119.70, 183.99 등)이며 품질보증·지속 업데이트 보장 없음. [커뮤니티]
