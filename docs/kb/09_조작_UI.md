## SAND: Raiders of Sophie — 조작 / 키바인드 / UI

> **신뢰도 최상위 요약 (가장 먼저 알아둘 것)**

### 1. 도보(On-Foot) 키바인드

아래 표는 ai-seo 위키 `sandraidersofsophiewiki.wiki`의 키보드 페이지 [AI위키-미검증]가 1차 출처이고, 마우스/WASD/Space/Ctrl/Shift/F/Tab/G/좌·우클릭 등 **굵게 표시한 공통 키는 다수 가이드·검색 스니펫이 교차 일치**하므로 신뢰도 중상이다.

| 동작 | 키 | 신뢰도 |
|---|---|---|
| **전진 / 후진 / 좌·우 이동** | **W / S / A / D** | 다수 일치 [커뮤니티·ai-seo] |
| **시점·자유 조준** | **마우스** | 다수 일치 |
| **점프(사다리·타워 플랫폼 포함)** | **Space** | 다수 일치 |
| **웅크리기** | **Left Ctrl** | 다수 일치 |
| **달리기(스프린트)** | **Left Shift** | 다수 일치 |
| **사격** | **마우스 좌클릭** | 다수 일치 |
| **조준(ADS)** | **마우스 우클릭** | 다수 일치 |
| 주무기 | 1 | ai-seo 추정 |
| 보조무기(권총류) | 2 | ai-seo 추정 |
| 멀티툴(Multi-Tool) | 3 | ai-seo 추정 |
| 재장전 | R | ai-seo 추정 |
| 근접 공격(밀리) | V | ai-seo 추정 |
| **수류탄 투척** | **G** | 다수 일치 |
| **상호작용(만능: 문·루팅·리액터·선장 테이블·탈출 호출)** | **F** (필요 시 길게 누르기) | 다수 일치 |
| 아이템 버리기 | X | ai-seo 추정 |
| 소비 아이템 빠른 사용 | Q | ai-seo 추정 |
| **인벤토리** | **Tab** | 다수 일치 |
| **지도** | **M** | ai-seo + 검색 스니펫 |

추가 메모:
- **인벤토리는 실시간(real-time)** — 여는 동안 게임이 멈추지 않는다(추출 슈터 특성). [AI위키-미검증, 장르상 타당]
- **패러쿠어/등반**: 개발진이 패러쿠어 애니메이션을 (재)추가·개선했다고 밝혔으나, **볼팅/맨틀에 별도 키가 있는지는 미확인**. 사다리·타워 등반은 자동 등반 + Space 점프로 보이며, 추출 타워 등반 시 트램플러에서 하차해 무방비 상태가 된다. [언론/커뮤니티]

### 2. 트램플러 조종(Driving / Helm)

핵심 흐름은 여러 출처가 일치한다: **① 갱웨이(E 또는 F)로 탑승 → ② 굴뚝(chimney) 아래의 전원/리액터 스위치를 길게 눌러(hold F) 가동 → ③ 조타륜(steering wheel / fly bridge)에서 상호작용 후 WASD로 조종.** 즉 조타석에 서면 이동키가 차량 조종으로 전환된다. [커뮤니티·언론 다수 일치]

| 동작 | 키보드 | 컨트롤러(추정) | 신뢰도 |
|---|---|---|---|
| **전원 켜기(리액터/메인 스위치)** | **F 길게 누르기** | 상호작용 길게 | 다수 일치 [언론·ai-seo] |
| 전진(가속) | W (1회=저속, **더블탭=가속**) | 좌스틱 ↑ (더블탭=가속) | GameRant [언론] |
| 후진 | S (계속 누름) | 좌스틱 ↓ | GameRant [언론] |
| 좌·우 선회 | A / D (1회=완만, **더블탭=급선회**) | 좌스틱 ←/→ (더블탭=급선회) | GameRant·nerdschalk |
| 측면 이동(스트래핑) | Q / E | LB·RB (Xbox) / L1·R1 (PS) | GameRant [언론] |
| **긴급 제동(브레이크)** | **Space** | B(Xbox)/O(PS) | GameRant [언론] — ⚠ 충돌 있음 |
| 헤드라이트/실내등 토글 | G | X(Xbox)/□(PS) | GameRant·nerdschalk |
| 경적(horn) | R 길게 누르기 | — | egamersworld·nerdschalk [ai-seo] — ⚠ 충돌 |
| 전원 끄기(주차 시) | 메인 스위치에 재상호작용 | 동일 | GameRant [언론] |

**출처 충돌(반드시 인게임 확인 필요):**
- 제동: GameRant는 **Space=긴급 제동**이라 하나, ai-seo 가이드들은 브레이크 키를 아예 언급하지 않음.
- 경적 R / 라이트 G: nerdschalk·egamersworld 주장. GameRant는 라이트를 G로 적어 **G=라이트는 일치**하나 경적 키는 불일치.

**전원/충전(조작과 직접 관련):** 트램플러는 가동 후 **약 10~12분**이면 전력이 소진된다. **NZ MK2 Energy Rod(에너지 로드)**를 인벤토리에 휴대 → 메인 스위치 근처 포트에 장착해 충전. 신규 캐릭터는 로드 5개로 시작(루팅·상점 보충). [언론·ai-seo 다수 일치, 수치는 EA-volatile]

### 3. 트램플러 에디터(빌드 모드) 키

`nerdschalk` 빌드 가이드 [ai-seo성 가이드 사이트]가 가장 구체적이다. **인게임 스크린샷으로 미검증**이므로 추정으로 취급할 것.

| 동작 | 키 | 신뢰도 |
|---|---|---|
| 새 블루프린트 생성 / 기존 데크 복사 | **C** (기존 데크 위에 호버 후 C로 빈칸에 복제) | nerdschalk·검색 일치 [ai-seo] |
| 선택 데크/파츠 제거 | **X** | nerdschalk [ai-seo] |
| 바닥(층) 뷰 올리기/내리기 | R / F | nerdschalk [ai-seo] |
| 카메라 회전 | Q / E | nerdschalk [ai-seo] |
| 줌 인/아웃 | 마우스 휠 | nerdschalk [ai-seo] |

- **에디터 탭 구성**: Chassis(섀시) / Essentials(필수) / Decks and Frames(데크·프레임) / Combat(전투) / Utility(유틸리티). [ai-seo 다수 일치]
- **필수 핵심 모듈**(없으면 경고/주행 불가): 섀시(토대) → 모터 리액터(동력) → **플라이 브리지/조타실(없으면 조종 자체 불가 — 1순위 실수)** → 입구(entrance) → 선장실(captain's compartment) → 표준 데크로 연결(경고가 사라질 때까지 C로 복사). [언론·ai-seo 일치]

### 4. UI 내비게이션 (인벤토리 휠 · 지도 · 쌍안경 · 추출 · HUD)

- **인벤토리 / 퀵 휠 = Tab.** 게임이 멈추지 않는 실시간 인벤토리. [AI위키 + 언론]
- **쌍안경(Binoculars)은 전용 키가 아니다.** **Tab을 누른 채(또는 인벤토리에서) 쌍안경 아이템을 선택**해 사용하며, 지평선·연기·접근하는 적 플레이어를 줌으로 확인한다. (TheGamer·Mobalytics 등 **[언론]**이 "holding tab and selecting the item"으로 명시 — 별도 단축키 설이 있다면 ai-seo 오류일 가능성.)
- **지도 = M** (토글). 반경 ~200m 내 무전 타워가 흰 원형 아이콘으로 표시. [검색 스니펫·ai-seo]
- **추출(Extraction)**: 무전 타워 콘솔에 **F 길게 누르기**로 호출 → **90초** 카운트 + 트램플러 상단에서 녹색 연막 분출, 집라인이 HUD에 표시. [언론 다수 일치]
- **HUD**: 무전 타워는 문 심볼이 있는 흰 원형 아이콘, 집라인 마커가 HUD에 표시됨. [언론]
- **F1 도움말(Help) 화면: 어떤 출처도 확인해주지 못함.** UE5 게임 관례상 존재할 수 있으나 **미확인** — 단정 금지.
- **핑/마커 전용 키**도 명시적으로 확인된 출처 없음 — 미확인.

### 5. 컨트롤러 지원 & Steam Input (가장 신뢰도 높은 영역)

- **[공식/primary]** tinyBuild 지원 FAQ: 게임은 **"full controller support(풀 컨트롤러 지원)"**. 단 **"Steam Input을 비활성화해야 하며, 그렇지 않으면 문제가 생길 수 있다"**고 명시. 개발진이 이 Steam Input 비호환을 **수정 작업 중**이라 함.
- **[커뮤니티/dev]** Steam 토론에서 개발자 **tinyBuild-Jay**가 "출시 시 컨트롤러를 지원한다"고 확정.
- **[커뮤니티 워크어라운드]** 네이티브 입력에 문제가 있을 때: *우클릭(라이브러리에서 게임) → 속성(Properties) → 컨트롤러(Controller) → Steam Input 활성화 → "컨트롤러 구성(Controller Configurator)"에서 키를 보거나 편집.* 단 다수 플레이어가 "Steam Input은 제대로 작동하지 않는다 / 거의 쓸 수 없다"며 **네이티브 입력 + Steam Input 끄기**를 선호.
- **[커뮤니티/핸드헬드]** Steam Deck / ROG Ally: 풀 컨트롤러 입력 지원, 백패들 **L4/R4를 점프·상호작용**에 매핑하면 보딩 전투 중 엄지를 스틱에 유지 가능. ProtonDB에 항목 존재.
- **진동(Vibration)**: 한 토론에서 진동 지원 질문이 있었으나 **공식 답변 없음 → 미확인.** 일부 ai-seo 위키가 "DualSense(PS5) 특화 진동 피드백 지원"이라 적었으나, **본 작품은 PC(Steam) 기준이라 PS5/DualSense 언급은 ai-seo 환각일 가능성이 높음 — 신뢰하지 말 것.** [AI위키-미검증, 의심]

### 6. 리매핑 · 설정 · 감도 · 데드존

- **키/마우스 리매핑 가능**: 설정은 **Esc → Settings → Controls**에서 접근, "모든 키바인드 커스터마이즈 가능"이라고 ai-seo 위키가 기술. [AI위키-미검증]
- **리매핑 메뉴 존재 자체는 커뮤니티가 확인**: "How to Change keyboard & Mouse Bindings", "How to Change Controls Settings" 같은 **유튜브 튜토리얼이 다수 존재** → 인게임에 키/마우스 재설정 UI가 있음. [커뮤니티]
- **컨트롤러 감도(Sensitivity) & 데드존(Deadzone) 슬라이더 존재**: "How to Adjust Controller Deadzone & Sensitivity Settings" 유튜브 튜토리얼로 **존재 확인**. 단 **정확한 수치 범위·기본값은 확인된 출처 없음(미확인).** [커뮤니티]
- **마우스 가속/마우스 인풋/조준 보조(aim assist)**: 개발 업데이트에서 **에임 어시스트·조준 개선** 작업이 언급됨(컨트롤러 대상). 마우스 가속 on/off 옵션 여부는 미확인(PCGamingWiki 입력 항목 미수집 — 403로 본문 확보 실패). [언론/미확인]

### 7. 미확인 · 주의 항목 정리

- ⚠ **개별 글자키(1/2/3, R, V, X, Q, Z, M)**: ai-seo 위키 단일 출처 중심 → 인게임 튜토리얼로 확정되지 않음.
- ⚠ **트램플러 브레이크(Space) / 경적(R)**: 출처 간 충돌.
- ⚠ **에디터 키(C/X/R/F/Q/E/휠)**: 단일 ai-seo 가이드 의존.
- ⚠ **F1 도움말, 전용 쌍안경 키, 전용 핑 키, 진동, 데드존 수치**: 확인 출처 없음.
- ⚠ **DualSense/PS5 진동**: ai-seo 환각 의심(PC 전용 작품).
- ✅ **가장 확실(공식)**: 풀 컨트롤러 지원 + "Steam Input 끄지 않으면 문제" (tinyBuild FAQ).
